commits.md 파일을 읽고, 이번 주 heg-wtf 조직의 활동을 요약하는 블로그 글을 생성해줘.

## 규칙

1. contents/ 폴더의 기존 글들을 참고해서 톤앤매너를 맞춰줘. 핵심은:
   - 비격식, 구어체, 1인칭 (나/내가)
   - 짧은 문단, 대화하듯 쓰기
   - 프로젝트별로 ## 소제목으로 구분
   - 솔직한 감정 표현 (고민, 아쉬움, 기대 등)

2. 커밋 메시지를 그대로 나열하지 말고, 의미 있는 주간 회고로 재구성해줘. 커밋에서 "무엇을 했는지"를 파악하고, 왜 그런 작업을 했는지, 어떤 방향으로 가고 있는지를 자연스럽게 풀어줘.

3. 오늘 날짜 기준으로 YYMMDD 형식의 파일을 생성해줘:
   - 디렉토리: contents/{YYMMDD}/
   - 파일명: {YYMMDD}.md
   - frontmatter 형식:
     ```
     ---
     title: '{YYMMDD}'
     author: 'heg'
     pub_date: 'YYYY-MM-DD'
     description: ''
     featured_image: ''
     tags: [커밋에서 파악된 프로젝트명들, 'heg']
     ---
     ```

4. tags에는 커밋에 등장하는 프로젝트/레포 이름을 한국어로 넣어줘 (예: '드리븐', '한우찾기', 'heg').

5. 분량은 기존 글들과 비슷하게 (200~500자 정도, 너무 길지 않게).

6. 커밋이 거의 없는 주에는 짧게 써도 괜찮아.
